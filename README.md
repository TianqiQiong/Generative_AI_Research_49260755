What does git clone do; what does github fork do.
Differences between git clone and github fork.

Git clone copies an existing Git repository from a remote location, such as GitHub, to your local computer.
A fork creates your own copy of someone else’s GitHub repository under your own GitHub account.
The main difference is that GitHub fork creates a copy of someone else’s repository under your own GitHub account, while git clone downloads a repository from GitHub to your local computer. Fork is mainly used when you want to contribute to a project that you do not own, because you can make changes in your own online copy and then submit a pull request to the original repository. Clone is mainly used when you want to work on a repository locally, edit files, commit changes, and push them back if you have permission.
